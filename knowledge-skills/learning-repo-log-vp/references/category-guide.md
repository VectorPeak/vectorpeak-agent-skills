# Learning Log Category Guide

Use categories to keep learning notes easy to scan. A log file may include these sections when it grows large:

```markdown
## 技术栈总览
## 概念记录
## 架构决策
## 工程实践
## 业务领域概念
## 易混淆点
## 待复习清单
```

## Category Rules

- **技术栈**: frameworks, libraries, databases, model providers, deployment tools
- **名词概念**: terms, algorithms, protocols, metrics, model names
- **架构决策**: why a project chooses one design over another
- **工程实践**: testing, logging, tracing, config, CI, refactoring, repo workflow
- **业务概念**: domain-specific words, user workflows, business rules
- **易混淆点**: concepts the user tends to mix up or compare
- **待复习问题**: open questions, TODO learning items, follow-up reading

## Default Initialization

When creating a new log file, start with:

```markdown
# {{project_name}} 学习复盘：技术栈与概念记录

这个文档用于记录当前项目开发和学习过程中需要长期复盘的技术栈、名词概念、架构决策、工程实践和业务理解。

## 技术栈总览

## 概念记录

## 架构决策

## 工程实践

## 业务领域概念

## 易混淆点

## 待复习清单
```
