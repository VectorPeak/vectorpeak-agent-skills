# Skill Maintenance Checklist

Each skill should be easy to inspect, install, and reuse.

- `SKILL.md` has a clear trigger and explicit boundaries.
- `README.md` explains the workflow for humans.
- References are scoped and named by purpose.
- Scripts are deterministic and avoid hidden network or credential assumptions.
- Examples are small enough to run during review.

This checklist helps keep the skill library useful as the number of skills grows.
